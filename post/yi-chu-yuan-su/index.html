移除元素

· 2024-09-30 · # 算法 # C语言

  • 理论
  • 代码
  • 总结

理论

题目大致是：给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并原地修改输入数组

当然，如果没有不要使用额外的数组空间，题目就会变得很简单（其实使用暴力破解也不难）；首先我们需要了解的这里移除的概念，移除并不是将他从内存中抹去，因为数组元素在内存地址中是连续的，不能单独删除数组中的某个元素，只能覆盖

总共有两种方法，一种是暴力破解法，一种是快慢指针法；暴力破解法就是两层for循环，一个用于遍历查找，一个用于更新新数组；快慢指针法是通过一个快指针和慢指针在一个for循环下完成两个for循环的工作，快指针是寻找新数组的元素 ，新数组就是不含有目标元素的数组，慢指针是用于指向更新新数组下标的位置，大致逻辑就是快指针一直遍历数组，同时慢指针也在同时加加，找到删除元素慢指针就停止加加一次，覆盖快指针的数据，也就是说慢指针才是处理后的数组

代码

int removeElement(vector<int>& nums, int val) {
        int slowIndex = 0;
        for (int fastIndex = 0; fastIndex < nums.size(); fastIndex++) {
            if (val != nums[fastIndex]) {
                nums[slowIndex++] = nums[fastIndex];
            }
        }
        return slowIndex;
    }

逻辑跟上述描述原理大差不差，如果不是删除元素，慢指针先是获取快指针数据，然后加加，如若不是，那么就跳过，同时也不会加，需要理解的是快指针是一直处于加加状态，最后返回慢指针；虽然这里只是返回了一个int，但实际数组已经被我们所改变了，所以得到慢指针后，从0开始到慢指针处遍历即是我们需要的数组

总结

需要强调的是，快指针是一直加加的，但慢指针不是，在是删除元素的时候会停止，以及这些指针并不是指针类型，而是一种泛称，是指向元素的意思，最后返回的值的意义也是重点，由于数组已经被我们修改了，所以通过慢指针处就是我们所求数组最后一个
